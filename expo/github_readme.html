GitHub Pages로 웹 페이지 만들기

  • 이전 글 ( GitHub Repository에 파일 업로드하기 )을 먼저 읽고 따라해봅니다.
  • 이전 글 ( GitHub Pages로 웹 페이지 만들기 )을 먼저 읽고 따라해봅니다.

github.com에 있는 본인의 정보문화기술입문 관련 Repoitory에 들어갑니다.

아래에 README.md 파일이 내용이 보입니다. 우측에 아이콘을 클릭합니다.

여기에서 README.md 파일의 내용을 수정할 수 있으며, .md 파일은 마크다운(markdown) 문법을 따릅니다.

markdown 문법에 대해서는 자세히 다루지 않겠습니다. 구글에 검색해 보세요.

여기에서는 GitHub Page 링크만 연결해보겠습니다.


            # jubilant-octo-winner

            * Link: <https://eunjs71.github.io/jubilant-octo-winner/>

내용을 작성한 후 Commit chages... 버튼을 누릅니다.

Repository에서 변경된 내용을 확인할 수 있습니다.

GitHub Desktop으로 돌아오면 Repository의 내용이 바뀌었기 때문에, Pull origin 버튼이 나타난 것을 확인할 수 있습니다.

(나타나지 않을 경우 상단의 Fetch Origin을 눌러보거나, 조금 기다려보세요.)

Pull origin 버튼을 누르면 로컬 환경(내 컴퓨터)에도 README.md의 변경 내용이 저장됩니다.

다른 팀원이 코드를 변경하고 Commit, Push 할때에도 Pull을 통해 해당 내용을 받을 수 있습니다.

README.md 파일을 정리해두면 추후에 많은 도움이 됩니다.
