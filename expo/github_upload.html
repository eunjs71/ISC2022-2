GitHub Repository에 파일 업로드하기

GitHub 회원 가입

  • GitHub 홈페이지에 들어가셔서 회원가입을 해주세요. → Go GitHub Page

새 Repository 만들기

New 버튼을 클릭하여 새 Repository를 만듭니다.

Repository 이름, 설명을 적습니다. Repository 이름은 나중에 GitHub Page를 만들 때 주소 이름이 되기도 합니다.

과제에 대한 설명까지 자세히 적고 싶으면, Add a README file을 체크합니다.

새 Repository가 생성되었습니다. 이제 GitHub Desktop이라는 응용프로그림을 통해 이 Repository를 관리할 것입니다.

GitHub Desktop 설치하기

  • GitHub Desktop 홈페이지에 들어가셔서 프로그램을 다운 받으세요. → Go GitHub Desktop Page

GitHub Desktop으로 Repository 관리하기

Preferece에 들어가 계정 로그인을 합니다.

Current Repository > Add > Clone Repository...를 선택합니다.

정기입 과제 업로드 용도로 만들어 둔 Repository를 원하는 위치에 Clone합니다.

해당 위치에 폴더가 생성된 것을 확인할 수 있습니다.

이제 파일을 복사합니다. index.html 파일이 있어야 나중에 GitHub Page를 만들 때 바로 연동됩니다.

제 파일들을 해당 폴더에 복사했습니다. 다시 GitHub Desktop으로 돌아갑니다.

Repository의 변경 내역을 확인할 수 있습니다.

우측하단에는 변경 내용을 적습니다.

변경된 내용을 설명을 포함에 묶는 것을 Commit이라고 합니다.

Description에는 상세 내용을 적습니다. 적지 않으셔도 됩니다.

내용을 기입했으면, Commit to main 버튼을 누릅니다.

Repository 안에 파일이나 파일의 내용이 변경(추가, 제거)되고 이를 commit 하면 다음과 같은 화면이 뜹니다.

Push origin 버튼을 누르면 github.com에 있는 Repository에 변경 내용이 적용됩니다.

이제 Push origin 버튼을 누르고 다시 github.com에 본인의 Repository에 들어갑니다.

추가한 파일이 업로드 된 것을 확인할 수 있습니다.

이후에도 변경된 사항이 있을 때마다, GitHub Desktop에서 commit > push를 통해 github.com에 있는 Repository에 변경 사항을 적용할 수 있습니다.
